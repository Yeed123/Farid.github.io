Project Submissions

Two projects applying lab knowledge to build complete Android applications.

Project 1
Navigation-Based Community App
Built on Lab 4 — Navigation with working features

Problem Statement

Urban residents often lack a centralized mobile platform to navigate local community services and information. This project addresses that gap by building a structured navigation-based Android app aligned with SDG 11.


Working Features

  • Multi-screen navigation with Bottom Navigation Bar
  • RecyclerView listing community resources
  • Detail view with relevant information
  • Intent-based screen transitions

Lessons Learned

Project 2
Cloud-Connected Smart Community App
Built on Lab 5 — Room + 2 Screens + API + Sensors + Cloud

Problem Statement

Community reporting and data collection in urban areas is fragmented. This project builds a sensor-aware, cloud-backed app that lets users log, store, and share local environmental or community data in real time.


Key Features

  • Room database for local persistent storage
  • 2 fully functional screens (List + Detail)
  • REST API integration for live data
  • Sensor integration (e.g. accelerometer / GPS)
  • Cloud sync (Firebase / backend)

Links

Watch VSR on YouTube
GitHub Repository

Lessons Learned

Gained experience integrating multiple technologies — local Room DB, live API calls, device sensors, and cloud storage — into a cohesive Android architecture.
